Implementing an Application

Applications should, ideally, be implemented alongside the creation of the corrsponding parameter file. Implementing an Application involves the following steps:

  1. Create the file: The file should be named <name>.py, where <name> is the name of the application.
    The file can be placed anywhere under the directory Code/application.

  2. Implement the Application class: You should import Application from application.application and create a class that extends it. Your class should implement the following instance methods:
    1. __init__: The constructor. Accepts a BaseParamSet (essentially the same as ParamSet, but you don't specify what section parameters should go in/come from) and uses it to initialize the Application.
    2. getReward: Receives a list choices of options that the algorithm is choosing to try as well as an (optional, there for backwards compatibility with influence maximization) set of chances that each option will be spontaneously influenced. The list of choices should expect the options to be identified in the same way in which they are identified within the listOptions function.

      This function should return the reward received from choosing that set of options.
    3. getOptionCount: This function returns the number of options available for an Algorithm to choose from.
    4. listOptions: This function returns a list of options available to an algorithm, with the identifiers that correspond to them within the system.
      For instance, if I have a set of of 13 options, identified with numbers starting from 1, but the option that was identified by 6 has been left out of the available options for some reason, then this method will return: [1, 2, 3, 4, 5, 7, 8, 9, 10, 11, 12, 13, 14].
    5. refresh: This method should do two things:
      1. It should re-initialize any variables that were initialized using randomness.
      2. It should reset any internal state that it may keep throughout an experiment run back to the starting state.
      If neither are relevant, than it can simply be implemented and do nothing.
  3. Write a 'createInstance' function: This function exists to act as a universal way to create an instance of the Application, it accepts a BaseParamSet object and passes it to the class's constructor, returning the class instance.
