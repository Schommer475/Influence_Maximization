Namespace Files

Namespace files are the files that are used by the namespace system to specify how the file paths for files produced running experiments should be structured.
Each potential configuration file also has a default setting that is used if that file does not exist.
All namespace configuration files are written as JSON files. (See here for a reference on JSON files.) Relevant files and folders mentioned in this guide may or may not currently exist inside the filesystem, depending on whether or not default values are being used for different configurations. Changing configurations through the alter_namespace.py interface will automatically create any folders and files needed, if you are creating the files here, you may also need to create the folders.

Section Ordering

  • description:

    This file determines whether it is application parameters or algorithm parameters that will appear first in file paths.

  • expected location:

    Files/Namespaces

  • expected file name:

    global_name_vars.json

  • default:

    The default configuration is for application parameters to appear before algorithm parameters.

  • fields:
    1. app_first:
      • type: boolean
      • description:

        If this field is set to true, application parameters will appear before algorithm parameters in the file paths. If it is set to false, algorithm parameters will appear before application parameters in file paths.

      • required: true

Application and Algorithm Files

  • description:

    These files are used to specify the experimental parameters that should appear in the file paths for output files of applications and algorithms as well as the order they should appear in and whether they should be joined together using basic separators or path separators.

  • expected location:

    Files/Namespaces/ApplicationNames or
    Files/Namespaces/AlgorithmNames
    depending on which one is being configured.

  • expected file name:

    <application/algorithm name>.json

  • default:

    The default configuration is to use the application/algorithm identifier and have it followed by a basic separator. No other parameters are used.

  • fields:
    1. headers:
      • type: list
      • description:

        List of elements to appear in the file path for files generated using the corresponding application/algorithm.

      • items:
        • type: string
        • description:

          The first item must be the application/algorithm identifier, any remaining items are names of experimental parameters of the application or algorithm, in the order that they should appear in. If the experimental parameter cannot be directly cast to a string, then it is the name of a variable holding a custom string representation of that parameter.

        • example: seed_set_size
      • required: true
      • Constraints: One or more. (The application/algorithm identifier must be present and must be the first item.)
    2. separators:
      • type: list
      • description:

        List of items indicating whether the element at the corresponding index of the headers list should be followed by a path separator or a basic separator in file paths.

      • items:
        • type: boolean
        • description:

          If true, the element at the corresponding index in the headers list will be followed by a path separator in the file path. If false, it will be followed by a basic separator.

      • required: true
      • Constraints: Same length as the headers field.

Joint Files

  • description:

    These files are used to determine what parameters, if any, will be used to identify files from individual runs of a given experimental setup. It also determines whether these parameters, if they are used, are a directory containing the files, or if they appear in the file name and are separated from the base file name by a basic separator character.

  • expected location:

    Files/Namespaces/JointNames

  • expected file name:

    <application name>_<algorithm name>.json

  • default:

    The default configuration is for a random ID to be used, no timestamp is used, and the random ID is joined to the base file name with a basic separator character.

  • fields:
    1. use_timestamp:
      • type: boolean
      • description:

        If this field is set to true, a timestamp with a precision level of seconds will be used to help identify individual runs of a particular experiment setup. The recommendation is to use this when you want to be able to easily identify and sort which batch of runs a result file came from.

      • required: true
    2. use_randID:
      • type: boolean
      • description:

        If this field is set to true, an eight digit random ID will be used to help identify individual runs of a particular experiment setup. The recommendation is to use this when you want to identify several parallel runs of the same experiment setup.

      • required: true
    3. sep_after:
      • type: boolean
      • description:

        The value of this field is only relevant when at least one of the two preceeding fields is set to true. When relevant, if this field is set to true, the value of the parameter(s) indicated by the previous two fields will become the name of a directory that houses the result files. If this field is set to false, then the value(s) will be joined to the base file name using a basic separator in order to form the full file name.

      • required: true
