run_experiment.py

run_experiment.py is a Python file used to run experiments that are specified using an input file. Note that this file expects to be run with the Code directory as the current working directory.

This function expects one argument, which must appear first, and an option which may optionally appear after the argument.

Argument and Option

  1. input file

    This is the path, relative to the current working directory, to the input file that should be used to specify experiments to run.

  2. -s <seed>:

    This option allows the user to specify a random seed that should be set before starting any part of the process to load and run the experiments. This is used to allow experiments to be reproducable and resumable by using a log of which experiments have and haven't completed. If this option is not provided, the system will randomly choose a seed.
    The seed the system is using will always be outputted to the user before beginning.

    Important note: The log files, located in Files/Temporary/Logs, are only used to be able to resume an interrupted set of experiments where they left off. For memory efficiency and to reduce the chance of randomly choosing a seed that already matches one on a log file, these files should be deleted every so often when you do not need them anymore.

Output Files

Each experiment iteration also produces an output file (location determined by the configuration of the namespace system) with a base file name of results.pkl. The file contains an object with two fields: params, contains a serialization of the experiment's ParamSet object and can be passed to ParamSet's fromDict method, and results, contains whatever data was returned from running the Algorithm's run method.
