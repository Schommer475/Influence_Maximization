Globals

Global variables used when running experiments, as well as their default values, are defined in Code/parameters/global_parameters.py
As of the time of writing, the variables currently available are:

  • num_cores:
    • type: integer
    • minimum value: 1
    • description:

      The number of processing cores to be used when running experiments and logging results.

    • default:

      The system will attempt to calculate the number of cpus available to it and use all of them.

  • num_runs:
    • type: integer
    • minimum value: 1
    • description:

      The number of times each experiment configuration is to be run.

    • default:

      1

  • refresh:
    • type: boolean
    • description:

      When a experiment specification is run more than one time in a given execution, it will use the same application and algorithm instances in every run. This means that randomized parameters, such as network weights, that are set at the time of creation will remain the same. Additionally, if the Application or Algorithm stores any internal state while running, that state may still remain between runs when running in single-threaded mode.

      This variable is passed to a function 'newRun' on the Applications and Algorithms, which, by default, will cause a user-defined refresh method to be called, where users can write any code necessary to reset the state and/or re-initialize randomized variables.

    • default:

      True
