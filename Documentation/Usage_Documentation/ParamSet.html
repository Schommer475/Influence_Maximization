ParamSet

ParamSet is a class designed to handle the storage and retrieval of parameter values relevant to an experiment.

Construction

The constructor for a ParamSet expects three BaseParamSet objects, one for the global parameters, one for the Application parameters, and one for the Algorithm parameters. The user of the system will almost never need to worry about this, because usually either the system will create the ParamSet for them, or they will create a ParamSet using the static fromDict method to create a ParamSet from a dictionary representation, usually one that has been loaded from a file.

Static methods

  • fromDict

    Parameters

    1. data: A dictionary representation of the paramset, usually created as a result of the toDict instance method

    Effect

    Takes a dictionary representation of a ParamSet and returns a ParamSet instance initialized from that data.

Instance methods

  • get

    Parameters

    1. section: An integer representing whether the parameter should be retrieved from among the application parameters, the algorithm parameters, or the joint parameters. This integer should be one of applications_index, algorithms_index, and joint_index, imported from Utilities/program_vars.py.
    2. name: The name of the parameter to retrieve.

    Effect

    Retrieves the parameter of the specified name from the specified section. If the section is invalid or a parameter of the given name does not exist in the specified section, an error will be raised.

  • reset

    Effect

    This class uses dynamically loaded instances of modules to look up any parameter values that were not specified by the user in the input file. However, Python is not able to pass module instances to sub-processes. This function temporarily clears out the module instances so that the ParamSet can be passed to a sub-process. The user should almost never have to worry about this, as the system handles this automatically when running experiments.

  • setAttr

    Parameters

    1. section: An integer representing whether the parameter to set should be among the application parameters, the algorithm parameters, or the joint parameters. This integer should be one of applications_index, algorithms_index, and joint_index, imported from Utilities/program_vars.py.
    2. name: The name of the parameter to set.
    3. value: The value to assign to the parameter.

    Effect

    Assigns the value to a parameter of the given name from the specified section. If the section is invalid, an error will be raised.

  • getApp

    Effect

    Returns the name of the application used in the experiment.

  • getAlg

    Effect

    Returns the name of the algorithm used in the experiment.

  • getPath

    Parameters

    1. timestamp: The experiment timestamp to possibly be used to identify the experiment in the file path. Usually passed in externally from the call to the run method.
    2. randId: The experiment random ID to possibly be used to identify the experiment in the file path. Usually passed in externally from the call to the run method, but can be manually generated inside the experiment run to differentiate between files from the same experiment run.

    Effect

    This function returns a string that represents an almost complete file path for output files generated by this experiment. Concatenating a base file name to the end of the returned string will yield a complete file name, assuming that the experiment is being run with the expected working directory.
    Assuming that the namespace system has been configured to use them for this Application/Algorithm combination, providing the same timestamp and/or random ID as those designated for the overall experiment run will group the file with other files from the same run (including the output file), while changing them will differentiate them from other files from the same run that may have the same base file name.

  • toDict

    Effect

    Returns a complete dictionary representation of the ParamSet. This representation can be stored in a file and then later passed into the static method fromDict to restore the ParamSet.
